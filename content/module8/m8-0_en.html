Introduction and Objectives

Welcome to Module 9 – Security Event Management

The Policy on Government Security defines a security event as “any event, act, omission or situation that may be detrimental to government security, including threats, vulnerabilities and security incidents.” A threat is defined as “any potential event or act, deliberate or unintentional, or natural hazard that could result in a compromise.” A vulnerability is defined as “a factor that could increase susceptibility to compromise.” A security incident is defined as “any event (or collection of events), act, omission or situation that has resulted in a compromise.”

It is important that you understand that it is your responsibility to report security events to your departmental security office. As an employee, YOU provide the “eyes and ears” on the ground to ensure that your workplace is secure. If you suspect a security event, report it!

Objectives

At the end of this lesson, you will be able to:

  • recognize a security event
  • summarize how to report a security event
